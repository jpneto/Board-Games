The WAG/XML project

by João Pedro Neto and Derek Hohls

Derek Hohls and me started a small project to create a standard format to translate game rules into XML markups. The goal is to accommodate the games presented at the World of Abstract Games (WAG) website into a data format that can be processed (in the future) by a database management system.

The main element is the tag <game> which includes every information concerning a specific game (and its variants).

<game>...</game>

Inside the <game> tag, there are some mandatory tags, namely:

<title>The game title</title>

<meta> [Information about the game] </meta>

The <meta> tag includes information not related to the game rules, like the game author and year, the publisher and the webpage writer. An example:

<meta>
   <year>1989</year>

   <author>
     <name>Author's Name</name>
     <mail>email@internet</mail> 
     <url>www.url.com</url> 
   </author>

   <publisher>
     <name>Ab Games, Lda.</name>
     <url>www.url.com</url> 
   </publisher>

   <writer>
     <name>Writer's name</name>
     <mail>email@internet</mail> 
     <url>www.url.com</url> 
     <year>2003</year>
   </writer>
</meta>

The <components> tag includes information about the game features except the rules. An example:

<components> 
  <board  top="5" middle="7" bottom="5">hex</board>

  <pieces number="14" color="black" id="soldiers" /> 
  <pieces number="14" color="white" id="soldiers" /> 

  <section>soldier</section>

  <features grid="hex" 
            cancapture="true" 
            canmove="true" 
            mustcapture="true"/>

  <zillions>http://www.zog.com</zillions>

  <diagram type="hex" middle="7">
       "5,7,5,wwwww/wwwwww/..www../..xxx../xxxxxx/xxxxx"
  </diagram>

  <image width="80" height="45" 
         src="icons/image.jpg" 
         text="a comment"/>

</components>

The <board> gives the typical size of the game board. The <pieces> describe the minimal physical requirements of game pieces in order to play the game. <section> provides information to what section the game belongs (soldier, king, stack). <features> denote some rule features (like pieces move, pieces capture, what type of grid, ...). <zillions> points to an optional link with ZoG information. The <diagram> draws a Javascript diagram of a certain board type (the type must be "sq", "hex" or "sqstack"). <diagram> may be used within other tags. The <image> inserts an image, it may be also use on other contexts. 

Next, comes the <description> tag, which includes the game rules and samples. The rules and divided in three parts: (i) the game setup, (ii) the rules where it's possible to create rule sections and rules subsections, and (iii) the game goal. An example:

<description>

  <setup>Setup description</setup>

  <rules>
    <rulesection>
       <rule><rtitle>MOVING</rtitle>rule 1</rule>
       <rule>rule 2</rule>
       <rule>rule 3</rule>
       <rulesection>
          <rule>rule 3.1</rule>
          <rule>rule 3.2</rule>
       </rulesection>
    </rulesection>
  </rules>

  <goal>Goal definition</goal>

  ...      plus zero or more examples
 
</description>

The format of an example is like the following two:

<example>
   <diagram type="hex" middle="7">  
      "5,7,5,.wwww/wwwwww/w.www../.axxx../.x.xxx/xx.xx"
   </diagram> 
   <title>An example</title>
   <note>it's possible to make <b>bolds</b>, newlines<br/>
         <i>italics</i>, <u>underlines</u>, urls <url>www...</url> 
         and images<br/><image width="80" height="45" src="a.jpg"
         text="a comment"/>
   </note>
</example>

<example>
  <diagram type="sqstack" col="5">   
    "4,5,..[..xWwx]../...../...../....."
  </diagram> 
  <title>Another example</title>
  <note>bla blabla blabla blabla bla</note>
</example>

There's also some optional tags for additional information.

<subtitle>The game subtitle</subtitle>   game subtitle

<notes>
  <note>General notes</note>
  <history>Some historical notes.</history>
  <rulenote>Rule notes</rulenote>
  <authornote>Author's notes</authornote>
</notes>

<related>                a related game
  <name>Blokus</name>
  <url>Blokus.htm</url>
</related>

<variant>                a variant
   <title>...</title>
   <subtitle>...</subtitle>
   <meta>...</meta>
   <components>...</components>
   <notes>...</notes>
   <description>...</description>
</variant>

Each <game> element is processed by the following game.xsl file using a transformation tool like Saxon. The format stuff is included at file wagstyle.css.

E.g., given a XML file abalone.xml, if we execute

java -classpath saxon.jar com.icl.saxon.StyleSheet 
     abalone.xml  game.xsl  >  abalone.html

we get abalone.html.

Some browsers, like IE6, can do this transformation automatically, so a link to the plain abalone.xml is enough.
